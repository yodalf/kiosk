## Workflow Preferences

**Git commit and push policy**
- NEVER commit or push without an explicit instruction from the user.

## What this project is

Single-file-ish shell project: a fullscreen video kiosk for Raspberry Pi that
drives `mpv` over an IPC socket to rotate through a playlist of YouTube URLs
with seamless transitions. Full design and runtime internals live in
**[ARCHITECTURE.md](ARCHITECTURE.md)** ŌĆö read that first for anything beyond
surface-level changes.

## Files in the repo

| File | Role |
|------|------|
| `kiosk.sh` | Runtime script. Spawns mpv, manages rotation, runs OCR highlight monitor. |
| `kiosk-setup.sh` | Idempotent installer. Copies scripts to `$HOME`, installs systemd units, configures autologin. |
| `kiosk-with-x.sh` | Template; real copy is generated by `kiosk-setup.sh` with the username hardcoded. |
| `kiosk.service` / `kiosk-monitor.service` | Systemd unit templates. `USERNAME` is substituted at install. |
| `kiosk-monitor.sh` / `restart-kiosk.sh` | Watchdog + restart helpers. |
| `99-vc4.conf` | Xorg config for the Raspberry Pi 5 dual-DRM GPU. |
| `*.url` | Playlists. `kiosk.url` is the active one; setup symlinks or copies. |
| `README.md` | User-facing quick start. |
| `ARCHITECTURE.md` | High-level overview down to `kiosk.sh` internals. Canonical source for how it works. |

## Deploy target

Live Pi at `realo@192.168.2.10` (or `raspi5.local`), password `toto`. The Pi
runs a copy of `kiosk.sh` in `~/` ŌĆö the repo is on the dev machine.

Deploy flow:
```bash
sshpass -p 'toto' scp -o StrictHostKeyChecking=no kiosk.sh realo@192.168.2.10:/home/realo/kiosk.sh
sshpass -p 'toto' ssh  -o StrictHostKeyChecking=no realo@192.168.2.10 'sudo systemctl restart kiosk.service'
```

For URL files (`*.url`), the running kiosk auto-detects edits to `~/kiosk.url`
within a few seconds ŌĆö no restart needed.

## When modifying `kiosk.sh`

- Bash 5+ is available on the target (uses `mapfile`, `shuf`, bash regex).
  macOS bash 3.2 won't run it ŌĆö test on the Pi via ssh.
- `stat` uses both `-f%z` (BSD) and `-c%s` (GNU) for portability.
- Log to `/tmp/kiosk.log` via `log_message` (prepends timestamp + rotates at 1 MB).
- The single `rotate_to_next "<reason>"` helper is the only path that should
  switch URLs. Don't reintroduce duplicated load/log/timer-reset sequences.
- Shuffle state is an in-memory bash array + index ŌĆö do not reintroduce the
  old `/tmp/kiosk_shuffle*` files.
- Live test: replace `~/kiosk.sh` on the Pi and `systemctl restart kiosk.service`,
  then tail `/tmp/kiosk.log` for ~30 s to catch startup, first rotation, and
  highlight monitor output.

## When modifying `kiosk-setup.sh`

- Must remain idempotent ŌĆö re-running on an already-set-up Pi must be safe.
- Must not run as root; it uses `sudo` only for system-level bits.
- If changing installed packages, update `apt-get install` and mention in
  ARCHITECTURE.md's platform notes.
